## 🔴 A-1 普查腳本(★ 拿到資料後第一個做,卡資料)

**目的**:燒 L40 前,用純 CPU 算出關鍵數字,決定「能不能做、做哪版」。

**要算的數字**:
- 可用縱向眼數(≥2 visit + 有治療 + QC 過)→ 預測訓不訓得起來
- 轉移對數(眼 × 配對)→ 訓練訊號量
- 積水消退事件數 → 要不要做存活頭
- 會變的眼比例 → persistence 招牌立不立得住
- 次數 vs 累積時間 相關係數 r → 「第幾針」claim 是否保守
- 治療分布(藥種/次數/naive 比例)
- 左右眼分布 → 病人切分 / fellow-eye
- (附)Δt 分布

**怎麼做**:純 numpy/pandas;重用 `latent_dynamics/baseline_cst.py`(算 CST)+ `stage1/longitudinal_dataset.py:summarize`(分眼/排序)。先 pilot 驗邏輯,再上全量。

**產出**:`census_report.json` + `census_report.md`(含決策表)。

**動工前置**:確認 `manifest.parquet` 欄位 + M7b `.npz` schema。

**規格**:`solutions/A1_census_spec.md`(已寫好,照它實作)。

## 拿到資料後的執行順序

1. 確認 schema → **寫 A-1 普查腳本** → 跑出 `census_report.md` 決策表(順便估 REG-1 QC 通過率)
2. 依決策表定版(存活頭?一步/多步?「第幾針」保守?)
3. 自標積水校準集(50-100 張,真 GT)

### ★ 最小版(用現成、不自己造,專測「預測成不成立」)
> **凍結 OCTCube(現成 3D,免蒸餾/adapter/JEPA/積水/REG-1)→ 一步治療條件預測器(殘差+change-weighted)→ 厚度頭(真 GT,先不做積水)→ 對照 persistence + 直接回歸。**
> 解耦「測預測」(用 OCTCube)和「造 FM」(蒸餾,之後加)。

### 存亡兩層(最小版要同時驗)
4. **P0:贏得了 persistence 嗎?**(會變的眼上)→ 贏不了 = 任務不可測 → **退 Phase 1 FM 論文**。
5. **L:latent 預測贏得了「直接回歸」嗎?** → 輸了 = 方法無理由 → 重新定位(賣標籤效率/多步/遷移)。
6. (之後)**空間 JEPA on/off(K)**:證明非 trivial 內插,沒比厚度-only 好就砍。

> 🔴 **洩漏鐵則**:病人層級切分**同時套到 Phase 1 時間相干性**(測試眼的縱向配對不進 Phase 1),不只 Phase 2。**很容易漏。**

### 核心站得住後,逐件加(每件消融閘門)
7. 2.5D 鄰窗 → 3D 聚合(空間 JEPA + 時間相干性)→ 第二三教師 → 多尺度解碼(O,消融)→ 積水 → REG-1 → 短程 rollout。
8. **Phase 1 凍結前閘門**:Track A 探測 **vs OCTCube**,贏才凍結。
9. Phase 2 完整評估(抗作弊 + 積水落真 GT + 病人切分 + 外部驗證)。

> 鐵律:先做最小版(2D 學生+一步預測+厚度頭)出數字 → 別一次全上。
